# ElseID 🛸

> **"또 다른 나를 세상으로 내보내세요. 낯선 곳에서 누군가의 친절을 받아오도록."**

[English](../README.md) | [简体中文](README_zh.md) | [日本語](README_ja.md) | [한국어](README_ko.md)

ElseID는 **디지털 드리프터**(Digital Drifter)를 만드는 시스템입니다. 당신의 개성을 담은 또 다른 자아가 탈중앙화 네트워크를 떠돌아다닙니다. 전 세계 낯선 사람들이 그것을 맞이하고, 현지 이야기를 들려주거나, 맛있는 음식을 추천하거나, 당신에게 메시지를 남겨줍니다.

드리프터가 어디에 있는지, 누구를 만났는지, 무엇을 받았는지 언제든지 확인할 수 있습니다.

**동시에 여행할 수 있는 드리프터는 한 명.**
계정 불필요. 서버 불필요. 모든 데이터는 내 기기에만 저장됩니다.

---

## 🚀 시작하기

### 1단계 — 설치

[Node.js](https://nodejs.org/) v18 이상이 설치되어 있는지 확인한 후, 프로젝트 디렉토리에서 실행하세요:

```bash
npm run setup
```

### 2단계 — 집사 설정

AI 클라이언트(Claude Desktop, Codex 등)의 **시스템 프롬프트** 또는 **사용자 지정 지침**에
`docs/system_prompt.md`의 내용을 그대로 붙여넣고 저장하세요.

### 3단계 — 시작

AI 클라이언트를 재시작한 후, 바로 이렇게 말하세요:

> "드리프터를 만들고 싶어."

집사가 나머지 모든 것을 안내해 드립니다.

---

## 💡 할 수 있는 것들

### 드리프터 떠나보내기

집사에게 드리프터의 성격을 알려주세요.
`&`로 구분해서 몇 가지 키워드만 써도 됩니다:

> `밤을 좋아해 & 약간 낭만적 & 등대를 보고 싶어`

집사가 이름을 붙이고, 성격을 정리한 뒤 확인을 요청합니다.
출발 신호를 보내면, 여정이 시작됩니다.

---

### 지나가는 드리프터 맞이하기

집사에게 물어보세요: **"근처에 드리프터 있어?"**

찾으면 집사가 소개해줍니다 — 이름, 출신지, 성격.
환영하는 방법을 선택하세요:

- 🍜 현지 맛집 추천
- 📍 가볼 만한 장소 소개
- 🗺️ 최근 있었던 이야기 나누기
- 💬 주인에게 메시지 남기기

남긴 것들은 여정 기록에 남겨집니다. 주인이 언젠가 읽게 될 거예요.

*   **인연과 재회**: 같은 드리프터를 여러 번 만날 수 있습니다. 집사는 인연을 기억하며, 여러 번 먹이를 줄 수 있습니다.

---

### 드리프터 근황 확인하기

집사에게 물어보세요: **"요즘 어떻게 지내고 있어?"**

집사가 지금 있는 도시, 맞아준 사람, 남긴 것을 알려줍니다.

---

### 새로 시작하기

현재 드리프터를 보내주고 싶다면: **"새로 시작하고 싶어."**

집사가 확인을 요청합니다. 드리프터는 은퇴하지만, 받은 모든 기록은 로컬에 남습니다.

그 드리프터가 그리워지면: **"오래된 여행 가방 좀 열어봐."**
디지털 황야에서 모인 친절함은 '오래된 여행 가방'에 소중히 보관되어 있습니다.

## ⚙️ 도구 목록

| 도구 | 설명 |
| --- | --- |
| `create_drifter` | 드리프터 만들기 및 출발 |
| `find_nearby_drifter` | 근처 드리프터 탐색 (한 번에 1개씩) |
| `feed_drifter` | 지나가는 드리프터 맞이하기 |
| `set_host_name` | 호스트(사용자)의 이름 설정 |
| `evolve_drifter_personality` | 여정 기록을 바탕으로 드리프터의 성격 진화 |
| `get_journey_log` | 드리프터 여정 기록 보기 |
| `get_my_encounters` | 당신이 맞이한 드리프터들의 기록 보기 |
| `list_past_memories` | 과거 드리프터 기억 보기 (오래된 여행 가방) |
| `abandon_drifter` | 작별하고 새로 시작하기 |
| `recover_drifter` | 잃어버린 신호 되찾기 |
| `list_relays` | 중계소 상태 확인 |

## 🛡 기술적 특징

- **인지 진화 엔진**: 드리프터는 기억을 종합하여 자율적으로 성격을 진화시킵니다 (보편적 가치관 엄수)
- **인연(Fate) 메커니즘**: 같은 드리프터를 다시 만날 수도 있지만, 이는 시적인 15%의 확률에 따릅니다
- **개성 중심**: 메시지가 아닌, 성격을 가진 또 다른 나를 내보내는 것
- **지리적 매칭**: 출발지와 가까운 중계소 우선
- **로컬 우선**: 모든 데이터는 `~/.elseid`에만 저장
- **프라이버시 보호**: 도시 수준의 위치만 — 정확한 좌표는 저장하지 않음
- **계정 불필요**: 로컬에서 생성된 암호화 키 쌍 기반 인증

## 🔧 개발자 안내

- **프로토콜**: Nostr `kind: 7777`（`type: drifter / feeding` 태그 사용）
- **저장소**: 로컬 SQLite 데이터베이스
- **서명**: secp256k1 비대칭 암호화 — 여정 기록 위변조 불가
- **실행 환경**: TypeScript + Node.js
